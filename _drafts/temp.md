# Ch1 從真實世界走入虛擬世界 - IP、MAC與網卡

IP是什麼？
如果你**房子的住址**是真實世界中的ID，那**IP**就是網路虛擬世界中的ID，你**必須透過這組ID才有辦法在網路世界寄信給其他人、跟其他人溝通**

網卡是什麼？
網卡是一塊被設計用來允許電腦在電腦網路上進行通訊的電腦硬體，這是維基百科上面的解釋
翻成白話文，它是電腦上的一塊零件，你必須要透過這塊零件才能連wifi（無線網路）、連有線網路等等；可以理解為讓你**從真實世界進入網路虛擬世界的橋樑**
每一片網卡都對應到一個**MAC位址**，也就是它的ID；可以理解為**那座橋的名字**（MAC位址理應上要獨一無二，但事實上是有可能重複；台灣有這麼多座橋，也一定**有幾座是有同樣的名字**）
這邊要補充一下︰連wifi的是一張網卡，如果你的電腦有可以接網路線的插孔，那它就會有另外一張網卡拿來連有線網路；連無線網路就要走橋A，連有線網路就要走橋B
再補充一點︰除了電腦，手機、路由器、網路孔轉USB的轉接頭上也都有網卡

MAC位址（實體位址）是什麼？
你要先理解網路卡（簡稱網卡）是什麼，然後MAC位址就是那塊網卡的ID

到這裡，你可以歸納出幾個重點：
* IP是網路虛擬世界中的ID
* 網卡是從真實世界進入網路世界的橋樑
* 網卡的MAC位址就像是橋的名字
* **一台裝置依不同的連線方式可能有多個網卡**（從真實世界進入網路世界的橋可以有很多座）
* **MAC位址是有可能重複的**(台灣有這麼多座橋，也一定有幾座是有同樣名字的)

# Ch2 網路世界的階層關係 - 路由器上的外網(WAN)與內網(LAN)

在網路世界中，你要找到別人、跟它溝通，必須**要有自己與他的IP**（自己的地址與他的地址），而**外網與內網的關係，就如同地球與地球上的所有國家**。假設地球是地球村，每個地球上的國家都有獨一無二的代碼，這個代碼就是外網的IP。而地球上的每個國家都有發行自己的代碼（身份證）用於識別自己的國民，這個「國家發行的代碼」就是內網IP。當然，如果你住的縣市也有發行自己的身份證，那這張「縣市發行的身份證」相對於「國家發行的身份證」就是內網與外網的關係。因此我們可以理解，內網與外網的關係是「相對」的。
回到溝通的部份，如果你要跟一個在國外的人溝通（這邊用寄信來比喻），你需要把信拿去縣市的郵局，縣市的郵局會把你的信轉給國家總郵局，國家總郵局會幫你寄給另一個國家，然後那個國家的總郵局又會把信往下分送，直到寄到那個人的手中。路由器就像郵局，負責轉送外網與內網的封包（信），每封信在轉送的過程中，代表身份的IP會不斷改變，如果是縣市郵局要寄給國家總郵局，它會用「國家發行的身份證」告訴國家總郵局這封信是誰寄的（因為這張身份證才能分辨是從哪個縣市寄出來的）；如果是國家總郵局要轉給另一個國家總郵局，它會用「地球發行的身份證」告訴另一個國家總郵局這封信是誰寄的（因為這張身份證才能分辨是從哪個國家寄出來的）

到這裡，你可以歸納出幾個重點：
* 外網與內網是相對的
* 每個人可以有「不同等級」的身份證
* 路由器就像是郵局，負責處理信件（訊息）的轉送
* 在「那個等級的郵局」，就要用「那個等級的身份證才能辨別」
